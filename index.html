Raster Tiles!

Raster Tiles

All tiles 256x256 pixels

Same grid, same boundaries

All these little tiles load way faster than one big map!

Seamless and fast

We call those slippy maps

Vergelijkbaar met TMS van OSGeo

Vergelijkbaar met WMTS REST van OGC

Zoom levels

Each zoom level has its own set of tiles!

Zoom level 0: 1 tile for the whole world.

Increases exponentially...

Zoom level 1: 4 tiles

Zoom level 2: 16 tiles

etc.

Map with tile bounds

Styling & Serving tiles

Disadvantages Raster Tiles

Rendering done before hand, tile set static and pre-defined

Storage size: multiple visualizations = multiple tile sets!

No interaction with objects, extra data needed

No object information, request to database needed

Advantages Vector tiles

Compressed data for fast data transfer.

Rendering is done on client side not by the server

High resolution!

Access to feature information!

So clients can style a map differently

Standards

Mapbox Vector Tile Standard. (MVT)

Based on the binary protocal buffer (.pbf) from Google.

Standard already used by Esri.

Formats

WEB GL

the rendering happens in your browser with the help of your device's graphics processing unit (GPU).

+ rotating and tilting the map

+ smooth/infinite zooming

WEB GL

WebGL is widely supported in modern browsers

OpenGL Shading Language for using the GPU - graphics processing unit
